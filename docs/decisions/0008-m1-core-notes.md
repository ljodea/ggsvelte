## Dependency choices (hand-rolled, no new deps)

1. **No d3-time / d3-time-format.** Time ticks are generated by a fixed
   interval table (s/min/h multiples, days, ISO-Monday weeks, month starts,
   1/2/5-stepped years) aligned to UTC calendar boundaries
   (`src/layout/time.ts`); labels use a hand-rolled strftime subset
   (`%Y %y %m %b %B %d %e %H %M %S %L %%`, deterministic en-US, UTC) plus
   d3-scale-style multi-scale default labels (year ticks "2026", month "Mar",
   day "Mar 05", time "14:30"). Rationale: the needed surface is ~150 lines,
   UTC-only, and byte-determinism is easier to audit than to configure.
   Revisit against the bundle-budget allowlist if locale-aware formatting is
   ever demanded.
2. **Numeric label formats are a documented d3-format-style SUBSET** —
   `d`, `,d`, `.Nf`, `,.Nf`, `.N%`, `s`, `~s` (`src/layout/format.ts`). The
   full d3-format grammar is deliberately not claimed; unknown format strings
   fall back to defaults with an `invalid-label-format` warning, never throw.
3. **Continuous color = a 10-stop viridis lookup table** with piecewise-linear
   sRGB interpolation (`src/scales/color.ts`, `VIRIDIS_RAMP_10`). Perceptual
   uniformity is inherited from the sampled stops, not re-derived. The plan's
   "simple 2-stop interpolation or a small viridis-like ramp table" — took
   the table. `scales.color.range` supplies custom ramp stops.
4. **Log ticks** are density-adaptive d3-log-style (1..9 mantissas when
   dense, 1/2/5, then bare powers of ten; exponent thinning by the 1/2/5 rule
   beyond one tick per decade).

## Design decisions

5. **Rule's two honest forms (Hadley lesson 15).** One `rule` geom, two
   signatures: annotation (fixed `params.xintercept`/`yintercept`) and
   data-driven (map exactly one of aes.x/aes.y). Annotation-form layers
   inherit **no plot aes** (normalize drops it — ggplot2's
   `inherit.aes = FALSE` on vline/hline); mixing forms, missing both, or
   mapping both axes are structured errors (`rule-form-ambiguous`/
   `rule-form-missing`/`rule-both-axes`). Annotation intercepts DO train the
   scales (ggplot2 parity).
6. **Bar/col x is always discrete (band).** ggplot2 supports continuous-x
   bars via `resolution()`; M1 treats every bar/col x as bands and emits a
   `bar-x-discretized` advisory for continuous fields. Continuous-x bars can
   land later without breaking the schema.
7. **Bars/areas use the FILL channel.** A color mapping on bar/col/area warns
   (`color-on-fill-geom`) and is ignored — in ggplot2, `color` on bars styles
   outlines, which the rects batch doesn't draw yet. Honest error over silent
   reinterpretation.
8. **Stacking order** = descending group id at the baseline, i.e. the
   first-seen group on top — identical to ggplot2 whenever first-occurrence
   order equals factor-level order (the R fixtures pin that shared case).
   DEVIATION: ggsvelte group ids are first-occurrence-canonical (decision
   0005), ggplot2's are factor-level-alphabetical, so unsorted data can stack
   in a different order than R. Same caveat applies to the count stat's
   output order.
9. **Dodge is per-x** (`preserve = "total"`, pinned against ggplot2 4.0.3):
   the groups PRESENT at an x split the 0.9 band evenly, ranked by ascending
   group id; an x with fewer groups gets wider bars.
10. **`zero: false` on a bar chart** suppresses the forcing advisory but the
    trained domain still reaches 0 — bar/area geometry contributes its zero
    baseline (`ymin`) to training, as in ggplot2. Excluding zero requires an
    explicit domain (and marks may then overflow the panel: continuous
    explicit domains do NOT censor marks in M1 — no panel clip yet;
    follow-up with coord work in M2). Pinned BAND domains do drop
    out-of-domain rows with a warning.
11. **`{ stat: "count" }` channel resolution** is implemented for `y` only;
    `{ stat }` on other channels warns (`stat-channel-unsupported`) and is
    ignored. Tier-2 validates stat-column names against the documented
    `STAT_COLUMNS` contract (identity → none, count → `count`).
12. **Tier-2 checks are opt-in** (`validate(spec, options?)`; any options
    argument, even `{}`, enables them). Tier 1 stays schema-shape-only so
    partial/composable specs keep validating; the pipeline enforces the same
    structural rules at render time with equivalent `PipelineError` codes.
    Rationale: the pipeline would otherwise re-scan inline data on every
    render. Limits: maxRows 100k, maxBytes 20MB (estimated from a 100-row
    sample), maxDepth 32, maxDiagnostics 100 — documented in
    `DEFAULT_VALIDATE_LIMITS`.
13. **Theme registry**: built-ins `default`/`light`/`dark`/`minimal` as a
    frozen table (no global mutable registry — Hadley lesson 14); object
    themes = named base + ink/paper/accent/grid role overrides. Roles feed
    geom defaults (ink → lines/points/rules/text, accent → bars/areas fills,
    paper → background). Every color renders as
    `var(--gg-<role>, <token>)`; the default theme's token is `currentColor`
    (M0c behavior preserved). Unknown names are tier-1 errors (schema literal
    union + `UnknownThemeError` for non-validating callers).
14. **Legends are measured BEFORE the two-pass layout** and folded in via the
    new `reserve` margins (together with the axis-title bands — this replaces
    the M0c post-layout +18px hack, decision 0007 follow-up 3). Ordering
    (`legend.order`: `stable-domain` default | `present-first-seen` |
    `sorted`) never reassigns colors. Both `color` and `fill` scales can
    produce legends (stacked vertically, right side).
15. **Ramp-legend gradient ids are deterministic** (`gg-ramp-<scale>`) in
    `renderToSVGString` — byte-determinism wins over cross-plot id
    uniqueness; inlining several SVG strings with ramp legends in one page
    can collide ids (documented caveat). The Svelte adapter uses
    `$props.id()` and is collision-free.
16. **CLI**: logic lives in `@ggsvelte/core` (`runCLI`, injectable IO,
    testable without spawning); the `ggsvelte-render` bin on the `@ggsvelte/svelte`
    package is a ~35-line wrapper. Exit codes: 0 rendered / 1 render failed
    / 2 usage error / 3 invalid spec. stdout carries ONLY SVG; diagnostics
    are stderr JSON lines ({kind: error|warning|advisory, ...}).
17. **Geom components are factory-built** (`createGeomLayer(geom, () =>
props, paramKeys)` over the live `$props()` proxy, read lazily inside
    descriptor getters) — each `<GeomX>` is ~25 lines.
18. **Ordinal palette fingerprints now include a scheme name**
    (`observable10` default) so `scales.color.range`/`scheme` changes
    invalidate stored assignments correctly. M0c states (range-hash
    fingerprints) invalidate once on upgrade — acceptable pre-0.1.0, noted
    here.

## R fixtures

`packages/core/tests/fixtures/positions/generate.R` (ggplot2 4.0.3, R 4.6.1)
generates 7 fixtures: count by x, count by x-and-fill, weighted count,
positive stack, mixed-sign stack, fill proportions, dodge offsets. Group
labels are carried through `aes(label = g)` because ggplot2's group id is the
x-by-g interaction for bars. Comparisons key on (x, g) so they are
row-order-independent. The `preserve = "total"` dodge behavior for uneven
groups was pinned by direct R experiment (test
"uneven groups divide each band among the groups PRESENT").

## Failure policy (implemented + tested)

- empty data (0 rows) → frame + axes placeholder, `empty-data` warning
- empty layer (no drawable rows post-stat) → skipped, `empty-layer` warning
- all-null mapped column → structured error (`all-null-column`, tier 2 and
  pipeline)
- zero-variance domain → symmetric padding (linear ±0.5, log ÷10/×10, time
  ±half day)
- log scale: non-positive data dropped with `log-nonpositive` warning;
  non-positive explicit domain refused (`log-domain-not-positive`)
- renderer exceptions → `PipelineError("renderer-failure")`, never blank
