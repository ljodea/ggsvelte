## Fixture coverage and achieved tolerances (the headline numbers)

All fixtures are generated by
`packages/core/tests/fixtures/stats/generate.R` (ggplot2 4.0.3, R 4.6.1)
through the public `layer_data()` API (bin/smooth/boxplot/summary) or
`stats::density()` directly, and asserted in
`packages/core/tests/stats-parity.test.ts`. Measured maxima, then the
asserted bound:

| Stat                                                                                                             | Reference                        | Measured max deviation                                              | Asserted       |
| ---------------------------------------------------------------------------------------------------------------- | -------------------------------- | ------------------------------------------------------------------- | -------------- |
| bin (7 cases: default 30, binwidth×boundary/center, closed left/right, edge values, weights, grouped+stacked)    | ggplot2 stat_bin                 | float noise (≤ ~1e-12)                                              | 1e-9           |
| lm fit/band/se (2 cases, levels .95/.99)                                                                         | ggplot2 stat_smooth(method="lm") | ≤ 2.2e-13 abs                                                       | 1e-9 rel       |
| loess fit/band/se vs `loess.control(surface="direct", statistics="exact")` (degree 2 span .75; degree 1 span .4) | stats::loess                     | ≤ 2.2e-13 abs                                                       | 1e-9 rel       |
| loess vs ggplot2's DEFAULT loess (surface="interpolate", statistics="approximate")                               | stats::loess defaults            | 0.53% (fit), 0.94% (band), 3.4% (se), relative to each column's max | 1% / 1.5% / 4% |
| boxplot (2 cases incl. coef=0.5)                                                                                 | ggplot2 stat_boxplot             | float noise                                                         | 1e-9           |
| summary mean_se + median/min/max                                                                                 | ggplot2 stat_summary             | float noise                                                         | 1e-9           |
| density (3 cases: nrd0, explicit bw + n=256, second group)                                                       | stats::density (binned FFT)      | 2.6e-4 rel; bw + grid endpoints exact                               | 5e-4 rel       |
| qt (28-point p×df grid)                                                                                          | R qt()                           | 5.4e-12 abs                                                         | 1e-9           |

**Loess, honestly:** ggsvelte implements the textbook local-regression
algorithm — the thing R computes with `surface = "direct",
statistics = "exact"` — and matches THAT to float noise, including σ, δ1,
δ2, and the t-based band (δ2 is computed exactly via dense n×n algebra for
n ≤ 300, and approximated by δ1 above; at those sizes qt barely moves).
ggplot2's default loess path instead _interpolates_ fits from a kd-tree of
vertices and _approximates_ the delta statistics — that gap, not ours, is
the sub-1% fit deviation documented above. Attribution for the algorithmic
lineage (SveltePlot → d3-regression → science.js / vega-statistics) is in
the repo NOTICE file; the R-parity rewrite shares structure (sorted window
walk, tricube), not code. R's gaussian family runs NO robustness
iterations, so the reference's robust reweighting was deliberately dropped.

## Seeded jitter (deliberate divergence from ggplot2)

`position: "jitter"` is ALWAYS seeded (mulberry32, `positionParams.seed`
default 42): the same spec renders the same pixels on every run and
platform. ggplot2 draws fresh random offsets per render unless a seed is
passed. This is a reproducibility-first choice — VR baselines, agent
round-trips, and renderToSVGString byte-determinism all depend on it — and
it is _surfaced_, not hidden: every jittered layer emits the
`jitter-seeded` advisory naming the seed and the override. Amount defaults
follow ggplot2 (40% of the data resolution, both axes; offsets are data
units on continuous scales, band-step fractions on discrete ones). Jitter
offsets do NOT widen scale training (a second deviation from ggplot2:
domains stay data-determined and deterministic).

## Other decisions and deviations

1. **Histogram is an alias** (plan's canonicalizer contract): the schema
   admits `geom: "histogram"` with its own discriminated params (agents DO
   emit it); `normalize()` rewrites it to `bar` + `stat: "bin"` + stacked,
   with `y: {stat: "count"}` filled (set `{stat: "density"}` etc. for the
   other bin columns). Equivalence tests compare normalized forms.
2. **Bin params live on BarParams** so the canonical bar+bin form stays
   schema-valid; `col` split off to its own ColParams (instance-compatible
   — every previously valid col layer still validates). `bins = 30` default
   emits the Hadley-lesson-12 advisory verbatim in spirit: "stat bin using
   bins = 30 … set binwidth or bins".
3. **Bin breaks are layer-wide** (from the layer's full x range, matching
   ggplot2's scale-dimension breaks) and every group × bin row is emitted,
   zero counts included — stacked histograms align because of both.
4. **Density grid**: R's `density()` semantics — n=512 grid over
   [min − cut·bw, max + cut·bw], cut default 3, bw.nrd0 — per GROUP.
   DIVERGENCE from ggplot2, which evaluates on the shared scale range and
   truncates the tails; the task pinned R's cut semantics and the fixtures
   pin `stats::density` directly. Kernel summation is DIRECT (exact) with
   an 8σ window cut (< 1.3e-14 error): R's binned FFT is the approximate
   one — the 2.6e-4 fixture gap is R's error, not ours. Weights normalize
   to sum 1 per group; `count` scales by unweighted n (ggplot2).
5. **Boxplot** dodges by default. ggplot2 4.x defaults to `dodge2`
   (slot-count-aware padding); ggsvelte uses the M1 dodge (per-x
   `preserve = "total"` slots) — visually equivalent for equal group
   counts, and one mechanism instead of two. Boxes compose from existing
   batch kinds: whisker segments (under), rects with the NEW optional
   stroke/fillRole (ink outline, paper fill when unmapped), median
   segments at 2× linewidth (ggplot2's fatten), outlier points. M2 scope:
   discrete x + quantitative y (tier-2 `channel-type-mismatch` otherwise);
   aes.weight unsupported (R delegates weighted quantiles to quantreg) —
   warns `weight-unsupported`.
6. **Smooth method inference**: omitted method → loess when the largest
   group < 1000 rows, else **lm** — ggplot2 escalates to mgcv::gam, which
   ggsvelte does not ship; the advisory says so explicitly. The se ribbon
   is a closed paths batch under the line, alpha 0.3 (ggplot2 uses 0.4 on
   grey60; 0.3 reads better over theme-accent/series tints), tinted by
   fill, else by the line's color scale, else theme accent. The band
   trains the y scale (never clips); `se: false` skips it.
7. **Summary stat**: mean_se is the default (fun "mean", no funMin/funMax
   → mean ± sd/√n, R fixture); explicit funMin/funMax (min/max/mean/
   median/sum) override. A non-mean fun without explicit bounds yields
   ymin = ymax = y — ggplot2 emits NA and draws nothing; drawing the
   zero-height bar is the honest visible version of "no spread defined".
8. **Errorbar** takes ymin/ymax aes channels (NEW, additive) with the
   identity stat, or computes them per x group with `stat: "summary"`.
   Identity-form position is identity-only this milestone (no dodge).
9. **positionParams** is a sibling of `position` (jitter width/height/seed,
   nudge x/y) rather than an object-valued position — keeps the position
   discriminator a plain literal for constrained decoding; the params
   object documents per-adjustment applicability per field.
10. **qt** is hand-rolled (Numerical Recipes inverse incomplete beta +
    Halley), 5.4e-12 vs R across the fixture grid. No new dependencies
    anywhere in M2 (`@sinclair/typebox` remains the only runtime dep).
11. **Error catalog growth** (all snapshot-tested): `computed-y-mapped`,
    `bin-center-and-boundary`, `channel-type-mismatch`; `bar-y-mapped` now
    covers histogram/bin. New warnings: `smooth-group-dropped`,
    `density-group-dropped`, `weight-unsupported` (+ `removed-missing`
    reused per stat's documented na-drop policy — every stat's module
    header states its missing-value policy, ggplot2 na.rm semantics: warn
    - drop). New advisories: `bin-default-bins`, `smooth-method-inferred`,
      `jitter-seeded`.

## Benchmarks (Apple Silicon dev machine, single-run pipeline cost)

- bin 100k rows: ~51 ms
- loess 5k rows + se band: ~380 ms (the exact-statistics pass is O(n·q),
  q = span·n — quadratic; **~25 ms without se**). 5k is the practical
  envelope for se bands; the method-inference cutoff (1000) keeps the
  default path far below it.
- density 100k rows: ~210 ms (after the 8σ window cut; 2.1 s naive).

Workloads are in `benchmarks/pipeline.bench.ts` (`histogram 100k`,
`loess smooth 5000 rows`, `density 100k`; bench-smoke runs reduced sizes).
